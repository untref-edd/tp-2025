# Material de Presentación Oral

Esta carpeta contiene el material para la presentación oral del trabajo práctico.

## Requisitos de la Presentación

- **Duración**: 10-15 minutos
- **Fecha**: 25 o 27 de Noviembre
- **Participantes**: Todos los integrantes del grupo deben participar

## Contenido Sugerido

### 1. Introducción (2 minutos)

- Presentación del equipo
- Descripción breve del problema
- Objetivos del trabajo

### 2. Enfoque Técnico (6-8 minutos)

- Explicación de las técnicas utilizadas:
  - Consulta a APIs (OpenAlex, The Lens)
  - Web Scraping (eventos y ferias)
  - Procesamiento RSS (noticias)
- Arquitectura del sistema
- Estructuras de datos aplicadas
- Desafíos técnicos y soluciones

### 3. Demostración (3-4 minutos)

- Ejecución en vivo del sistema
- Mostrar la interfaz de consola
- Ejemplo de consulta a cada fuente
- Visualización de archivos CSV generados

### 4. Resultados y Conclusiones (2 minutos)

- Resultados obtenidos
- Lecciones aprendidas
- Trabajo en equipo
- Posibles mejoras

### 5. Q&A (variable)

- Preguntas del docente y audiencia

## Estructura Recomendada de Archivos

```text
presentacion/
├── README.md (este archivo)
├── slides.pptx (o .pdf, .key, .odp)
├── demo/ (scripts para demostración)
│   └── demo.sh (script de demostración)
└── recursos/ (capturas, videos, etc.)
```

## Herramientas Sugeridas

- **Google Slides** (colaborativo, online)
- **Microsoft PowerPoint**
- **LibreOffice Impress** (gratuito)
- **Canva** (diseño atractivo)
- **reveal.js** (HTML, para técnicos)

## Consejos

1. **Practicar la presentación** varias veces antes
2. **Cronometrar** para asegurar que no excedan 15 minutos
3. **Preparar respuestas** a posibles preguntas técnicas
4. **Tener backup** de la demostración (video o capturas)
5. **Todos participan**: distribuir partes de la presentación
6. **Claridad**: explicar conceptos técnicos de forma comprensible
7. **Destacar aprendizajes**: más allá del código, qué aprendieron

## Checklist Pre-Presentación

- [ ] Slides completos y revisados
- [ ] Ensayo completo realizado
- [ ] Demo probada y funcional
- [ ] Backup de demo preparado
- [ ] Tiempo verificado (10-15 min)
- [ ] Roles asignados a cada integrante
- [ ] Material accesible (USB, cloud, email)
- [ ] Código actualizado en el repositorio
