# Documentación del Proyecto

Esta carpeta contiene toda la documentación técnica del trabajo práctico.

## Estructura

- `analisis/` - Análisis y diseño de cada técnica implementada
- `diagramas/` - Diagramas de arquitectura, flujo y clases
- `presentacion/` - Material para la presentación oral (10-15 minutos)
- `notebook/` - Jupyter Notebooks con el informe técnico

## Entregables

### 1. Informe en Jupyter Notebook (.ipynb)

El informe debe incluir:

- **Análisis y diseño** de cada técnica (APIs, web scraping, RSS)
- **Explicación de librerías** utilizadas y justificación
- **Descripción de estructuras de datos** aplicadas
- **Problemas encontrados** y cómo se resolvieron
- **Conclusiones y aprendizajes** del grupo
- **Archivos CSV generados** y análisis de los datos

### 2. Presentación Oral

Material para la presentación de 10-15 minutos que debe cubrir:

- Descripción del problema resuelto
- Enfoque técnico adoptado
- Demostración del sistema funcionando
- Resultados obtenidos
- Lecciones aprendidas

### 3. Documentación Técnica

- Arquitectura del sistema
- Diagramas de componentes
- Análisis de complejidad
- Guía de uso

## Roles y Responsabilidades

Cada integrante del grupo debe contribuir a la documentación según su rol específico.
Todos los integrantes deben participar en el informe y en la presentación.
