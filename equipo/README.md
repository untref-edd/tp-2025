## Normas de Colaboración

1. **Commits identificables**: Cada integrante debe realizar commits frecuentes con su nombre
2. **Mensajes descriptivos**: Usar mensajes de commit claros y descriptivos
3. **Branches por feature**: Usar ramas separadas para cada funcionalidad nueva
4. **Code review**: Revisar código antes de merge a main
5. **Documentación**: Documentar cada función y módulo con docstrings
6. **Reuniones semanales**: Coordinar avances y resolver bloqueos
7. **Testing**: Escribir pruebas para cada módulo desarrollado
